# Airadio — build spec (hackathon: Stanford x DeepMind, Replit track)

One-liner: **Airadio turns your interests, today's news, and your music taste into a live personal radio show.** Gemini writes the show, Gemini TTS voices an original host ("Nova"), music comes from bundled royalty-free tracks with radio-style crossfades.

Stack: Next.js 16 App Router, TypeScript, Tailwind v4 (utility classes only), `@google/genai` v2.12 (already installed). Node runtime for all API routes (they use `fs`). Repo root: this directory. Shared types: `lib/types.ts` — import from `@/lib/types`, never redeclare.

## Env vars (.env.example documents all; .env.local is gitignored)
- `GEMINI_API_KEY` (required for live mode)
- `GEMINI_TEXT_MODEL` default `gemini-2.5-flash`
- `GEMINI_TTS_MODEL` default `gemini-2.5-flash-preview-tts`
- `GEMINI_TTS_VOICE` default `Puck`
- `NEXT_PUBLIC_GOOGLE_CLIENT_ID` optional — absent ⇒ UI shows Demo Mode only

## Hard rules
- **Never fail the demo.** Every Gemini call has a hardcoded, decent-quality fallback (canned show script, canned chips). No route ever 500s because a key is missing — degrade to fallback and set `"fallback": true` where the contract allows extra fields.
- No audio download/extraction from YouTube, no watch-history claims. YouTube Data API (client-side, user's OAuth token) reads **liked videos + subscriptions** only.
- Host is an ORIGINAL persona ("Nova" — warm, quick-witted, slightly nocturnal). Never imitate a real person. Ads always start with an explicit sponsor disclosure ("Today's show is brought to you by …"). Reject/skip campaigns with medical/financial/gambling/political content — Gemini prompt must say to use `facts` verbatim-safe claims only.
